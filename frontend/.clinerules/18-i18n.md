# Internationalization (i18n)

## Core Rules
1. **No hardcoded user-facing strings** — All text must use translation keys
2. **No string concatenation** for translations — Use interpolation
3. **Dates, numbers, currency** must use locale-aware formatting

## i18n Library
Use `react-i18next` (or the project's chosen i18n library):

```typescript
import { useTranslation } from 'react-i18next';

const UserProfile: React.FC = () => {
  const { t } = useTranslation();

  return (
    <div>
      <h1>{t('userProfile.title')}</h1>
      <p>{t('userProfile.welcomeMessage', { name: user.name })}</p>
    </div>
  );
};
```

## Translation Key Naming

### Convention: `namespace.section.element`

```json
{
  "common": {
    "buttons": {
      "save": "Save",
      "cancel": "Cancel",
      "delete": "Delete",
      "confirm": "Confirm"
    },
    "labels": {
      "name": "Name",
      "email": "Email",
      "status": "Status"
    },
    "messages": {
      "loading": "Loading...",
      "noData": "No data available",
      "error": "An error occurred"
    }
  },
  "userProfile": {
    "title": "User Profile",
    "welcomeMessage": "Welcome, {{name}}!",
    "lastLogin": "Last login: {{date}}"
  },
  "validation": {
    "required": "{{field}} is required",
    "minLength": "{{field}} must be at least {{min}} characters",
    "invalidEmail": "Please enter a valid email address"
  }
}
```

### Rules
- Use `camelCase` for keys
- Group by feature/page namespace
- Use `common` for shared translations
- Use `{{variable}}` for interpolation (not concatenation)
- Keep keys descriptive but concise

## ❌ Forbidden

```typescript
// ❌ Hardcoded strings
<button>Save</button>
<p>Welcome to the dashboard</p>
<span>Error: Something went wrong</span>

// ❌ String concatenation
const message = 'Hello, ' + userName + '! You have ' + count + ' messages.';

// ❌ Template literals for translations
const message = `Welcome ${userName}`;
```

## ✅ Correct

```typescript
// ✅ Using translation keys
<button>{t('common.buttons.save')}</button>
<p>{t('dashboard.welcome')}</p>
<span>{t('common.messages.error')}</span>

// ✅ Interpolation
const message = t('dashboard.greeting', { name: userName, count: messageCount });
```

## Date & Number Formatting

```typescript
// ✅ Locale-aware date formatting
import { format } from 'date-fns';
import { ko, enUS } from 'date-fns/locale';

const formattedDate = format(new Date(), 'PPP', { locale: currentLocale });

// ✅ Number formatting
const formattedNumber = new Intl.NumberFormat(currentLocale).format(1234567.89);

// ✅ Currency formatting
const formattedCurrency = new Intl.NumberFormat(currentLocale, {
  style: 'currency',
  currency: 'KRW',
}).format(50000);
```

## Pluralization

```json
{
  "items": {
    "count_zero": "No items",
    "count_one": "{{count}} item",
    "count_other": "{{count}} items"
  }
}
```

```typescript
t('items.count', { count: itemCount });
```

## Translation Workflow
1. Add keys to English JSON first (default language)
2. Use descriptive key names
3. Add Korean translations
4. Test both languages in UI
5. Review with native speakers
